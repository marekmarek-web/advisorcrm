# UX/UI polish 2026-04-20 — Revolut-style design system

Datum: 2026-04-20 · Rozsah: mobile-first redesign portálu + nový design-system layer.

## Motivace

Uživatel hlásil, že nativní vzhled několika screenů působil „AI-slop“ — nekonzistentní
paddingy, ad-hoc barvy, english enum hodnoty v UI (`member`, `First Name`), syrová ID
(`label_17762…`) a zlomené mobilní stavy (kalkulačka blokovaná dockem, produkce padala na
chybějící SQL sloupec). Cílem bylo narovnat to na úroveň moderních SaaS 2026 (Revolut, Linear,
Attio): jednotné hero karty, měkčí rádius, méně grafického šumu, větší hit-area a jasný
typografický rytmus.

Redesign byl schválen jako **deep** — nové design-system primitives + přepis dotčených screenů
na ně (ne jenom tweak hex barev).

- `--aidv-hero-radius: 24px` / `--aidv-card-radius: 20px` / `--aidv-pill-radius: 16px`
- `--aidv-shadow-card-sm/-md/-hero` (progressive elevation)
- `--aidv-metric-ok / -warn / -crit` (health barvy)

## Guidance pro další screeny

Když budeš další screen přepisovat na „Revolut 2026“ styl, postup:

1. **Hero nahoru**: `<HeroCard eyebrow="Sekce" title="…" meta={…} actions={<HeroAction>…</HeroAction>}>…</HeroCard>` místo lokálního tmavého div gradientu. Nepouštět barvy ze specu — použít `tone="navy"` (default), `tone="indigo"` pouze pro positiv/celebrate stavy.
2. **KPI vždy přes MetricGrid + KpiCard**. Progress bar získáš zdarma pokud pošleš `target`. `health` chipa je *optional* — zapínej ji jen když má opravdu význam (SLA / risk / target gap), ne pro kosmetiku.
3. **Periody, tab lišty, scope → `SegmentPills`**. `FilterChips` nech jen pro „wrap of many filters“, kdy očekáváme overflow.
4. **Chybové a empty stavy → `InlineAlert`**. Fullscreen `ErrorState` použij pouze pokud je screen totálně nevyužitelný (nenačtená auth, 403 atd.).
5. **Modály a sheety → `SheetChrome` pattern**. Header 40×40 close, px-5 py-4, subtitle do `truncate`, body `overflow-y-auto p-5 sm:p-6`, `max-h-[85vh]` na mobilu.
6. **Žádné hard-coded hex barvy** — jedem přes tokeny (`--wp-text`, `--aidv-hero-navy`, `--aidv-metric-ok`). Nové barvy = nejdřív token, pak teprve použití.
7. **Hit-area min 40 px**. Close tlačítka 40×40, pill tlačítka `min-h-[36px]` (sm) / `min-h-[44px]` (primary CTA na mobilu).

## Neudělané / odložené

- **Desktop `BusinessPlanView.tsx`** a **desktop `TeamOverviewView.tsx`** — větší rework rámce, drží se původní layout. Primitives jsou nachystány; příští iterace může sebrat HeroCard do hlavičky a přepnout KPI na MetricGrid.
- **Studené kontakty** — samotný screen zůstává intakt, jen je ukrytý feature flagem. Až se bude znovu zapínat, projít ho stejným rytmem (HeroCard + InlineAlert pro empty states).
- **Kalkulačka Investment/Life** — výsledkový dock má jen generický collapse, mohl by dostat `collapsedSummary` (např. „1,2 M Kč · 10 let“) při dalším průchodu.
